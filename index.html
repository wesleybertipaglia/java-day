Domine a complexidade e os desafios da integração NoSQL em aplicações Java cloud-native

Maximillian Arruda

Hoje em dia, é normal ver soluções cloud-native lidando com uma grande quantidade de informações. Nesse cenário, os bancos de dados NoSQL estão ganhando uma posição de grande relevância, mas lidar com muitas soluções NoSQL de muitos fornecedores diferentes é muito difícil. Em termos de lidar com bancos de dados relacionais, não há dúvida de que a especificação JPA é uma API bem estabelecida no mundo Java, mas e se houver uma forma bem definida, bem parecida com JPA, de trabalhar com NoSQL com Java? A especificação Jakarta NoSQL foi criada para facilitar a experiência do desenvolvedor ao trabalhar com multiplos bancos de dados NoSQL em suas aplicações Java de maneira fácil e produtiva. Nesta palestra, através de uma sessão live-coding vamos abordar como utilizar a Jakarta NoSQL para dominar a integração com banco de dados NoSQL em suas aplicações java cloud-native.
